che improvements

i want to have 2 kinds of logs, for human consumption when CLI is used, and for machine consumption (e.g. when otel and prometheus are involved)

general approach would be if debug is not set we only print information about what happened
  if debug is set we also add info what is going to happend and what is not going to happen and why

I want to resign from label(sth, sth) form of log

e.g. in discover-profiles in non debug important information to show is:
  che spec path
  remotes initialized/updated (and which) and that they landed in cache (abbreviate path), one per remote
  profiles discovered with their ops
    and what changes are going to be made for each op (delta only)
    and each profile working directory

in debug additional info to be shown:
  config (but only delta from defaults, do not show full)
    there should be che subcommand che print-config delta, che print-config all to output this information

  profiles that failed discovery with reason (no ops needed)

in CLI, there is no need for each log to be in one line
  our goal is to make it readable for a user, we can use headings, indentation etc.
